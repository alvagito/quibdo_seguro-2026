DESCRIPCIÓN GENERAL
-------------------
"Quibdó Seguro" es una plataforma web que permite a ciudadanos reportar
incidentes de seguridad (robos, accidentes, violencia), participar en un
sistema de puntos/recompensas, y facilita la gestión de reportes por parte
de autoridades y administradores.

STACK TECNOLÓGICO
-----------------
- Backend: Laravel 10.10 + PHP 8.1+
- Base de datos: MongoDB (NoSQL) via mongodb/laravel-mongodb ^5.5
- Frontend: HTML5, CSS3, JavaScript Vanilla
- Mapas: Leaflet.js + OpenStreetMap
- Iconos: Font Awesome 6
- Build tool: Vite 5.0
- Arquitectura: MVC (Modelo-Vista-Controlador)

MODELOS (app/Models/)
================================================================

User.php
  Campos: nombre, email, password, rol, puntos
  Roles posibles: 'normal' (ciudadano), 'admin', 'autoridad', 'comercio'
  Relaciones:
    - hasMany(Incidente)
    - hasMany(Oferta)
    - hasMany(Notificacion)
    - hasMany(Canje)
    - hasMany(Comentario)

Incidente.php
  Campos: id_usuario, id_tipo_incidente, latitud, longitud, descripcion,
          fecha_hora_incidente, fecha_hora_reporte, evidencia_foto_url,
          direccion_aproximada
  Tipos: 1=Robo, 2=Accidente, 3=Violencia, 4=Otro
  Relaciones:
    - belongsTo(User)
    - hasMany(Comentario)

Oferta.php
  Campos: id_comercio, titulo, descripcion, puntos
  Relaciones:
    - belongsTo(User) [el comercio]
    - hasMany(Canje)

Canje.php
  Campos: id_usuario, id_oferta, id_comercio, puntos_canjeados,
          codigo_qr, estado, fecha_canje, fecha_validacion
  Estados: 'pendiente', 'validado'
  Relaciones:
    - belongsTo(User) [usuario que canjea]
    - belongsTo(Oferta)
    - belongsTo(User) [comercio que valida]

Recompensa.php
  Campos: nombre, puntos
  Uso: Recompensas especiales creadas por administradores

Notificacion.php
  Campos: id_usuario, tipo, titulo, mensaje, leida, url, datos_adicionales
  Tipos: 'validacion_reporte', 'nuevo_reporte', 'canje'

Comentario.php
  Campos: id_incidente, id_usuario, comentario, es_autoridad
  Relaciones:
    - belongsTo(Incidente)
    - belongsTo(User)

CONTROLADORES (app/Http/Controllers/)
================================================================

AuthController.php
  showLogin()     → Muestra formulario de login
  login()         → Procesa credenciales, redirige según rol
  showRegister()  → Muestra formulario de registro
  register()      → Crea usuario con rol='normal' por defecto
  logout()        → Cierra sesión y regenera token CSRF

DashboardController.php
  index()         → Panel principal con feed de noticias y estadísticas
                    Datos: incidentes recientes (últimos 20), total_reportes,
                    reportes_hoy, reportes_semana, usuarios_activos,
                    incidentes por tipo

UsuarioController.php
  perfil()                    → Perfil del usuario
  formReportar()              → Formulario para reportar incidente
  mapa()                      → Mapa interactivo con filtros (tipo, periodo)
  recompensas()               → Lista recompensas y ofertas disponibles
  canjearRecompensa($id)      → Canjea puntos por oferta, genera código QR,
                                resta puntos, crea registro de canje
  misCanjes()                 → Historial de canjes del usuario
  notificaciones()            → Lista notificaciones del usuario
  marcarNotificacionLeida()   → Marca notificación como leída
  guardarReporte()            → Guarda incidente, valida ubicación (radio 100m),
                                sube foto, suma puntos, notifica autoridades

AdminController.php
  index()                     → Dashboard admin
  usuarios()                  → Lista usuarios con búsqueda y filtro por rol
  editUsuario($id)            → Formulario de edición de usuario
  updateUsuario()             → Actualiza datos del usuario
  deleteUsuario()             → Elimina usuario
  reportes()                  → Lista reportes con búsqueda y filtros
  deleteReporte()             → Elimina reporte
  comercios()                 → Lista comercios con sus ofertas
  ofertas()                   → Gestiona ofertas de comercios
  editOferta() / updateOferta() / deleteOferta()
  recompensas()               → Gestiona recompensas del sistema
  storeRecompensa() / deleteRecompensa()

ComercioController.php
  index()                     → Dashboard del comercio
  ofertas()                   → Lista ofertas del comercio
  storeOferta()               → Crea nueva oferta
  deleteOferta()              → Elimina oferta
  estadisticas()              → Métricas: total_ofertas, canjeadas,
                                puntos_redimidos, canjes_pendientes
  validarCanje()              → Valida código QR, marca canje como 'validado',
                                registra fecha_validacion

InicioController.php
  index()                     → Página pública con mapa, últimos 20 incidentes,
                                últimos 5 reportes, estadísticas generales

BASE DE DATOS - COLECCIONES MONGODB
================================================================

Colección: usuarios
  _id, nombre, email, password (bcrypt), rol, puntos, created_at, updated_at

Colección: incidentes
  _id, id_usuario (ref), id_tipo_incidente (1-4), latitud, longitud,
  descripcion, fecha_hora_incidente, fecha_hora_reporte,
  evidencia_foto_url (nullable), direccion_aproximada, created_at, updated_at

Colección: ofertas
  _id, id_comercio (ref), titulo, descripcion, puntos, created_at, updated_at

Colección: canjes
  _id, id_usuario (ref), id_oferta (ref), id_comercio (ref),
  puntos_canjeados, codigo_qr (unique), estado ('pendiente'|'validado'),
  fecha_canje, fecha_validacion (nullable), created_at, updated_at

Colección: recompensas
  _id, nombre, puntos, created_at, updated_at

Colección: notificaciones
  _id, id_usuario (ref), tipo, titulo, mensaje, leida (bool),
  url (nullable), datos_adicionales (object), created_at, updated_at

Colección: comentarios
  _id, id_incidente (ref), id_usuario (ref), comentario,
  es_autoridad (bool), created_at, updated_at

FLUJOS PRINCIPALES DEL SISTEMA
================================================================

1. AUTENTICACIÓN
   Usuario → /login → Valida credenciales
   → Redirige según rol:
     admin     → /admin
     comercio  → /comercio
     normal    → /dashboard
     autoridad → /dashboard (con vistas especiales)

2. REPORTE DE INCIDENTE
   Usuario → /reportar → Selecciona ubicación en mapa Leaflet
   → Completa formulario → POST /reportar
   → Valida datos → Sube foto de evidencia
   → Crea Incidente en MongoDB
   → Suma 10 puntos al usuario
   → Notifica a todas las autoridades
   → Redirect /dashboard

3. CANJE DE RECOMPENSA
   Usuario → /recompensas → Selecciona oferta
   → POST /recompensas/canjear/{id}
   → Valida que tenga suficientes puntos
   → Genera código QR único
   → Crea registro Canje (estado='pendiente')
   → Resta puntos al usuario
   → Notifica al usuario
   → Muestra código QR

4. VALIDACIÓN DE CANJE (por comercio)
   Comercio → /comercio/estadisticas → Ingresa código QR
   → POST /comercio/validar-canje
   → Busca Canje por codigo_qr
   → Si existe y estado='pendiente':
     → Marca como 'validado'
     → Registra fecha_validacion
   → Redirect con confirmación

NOTAS IMPORTANTES PARA CHATGPT
================================================================

1. El proyecto tiene DOS versiones:
   - Versión Laravel (carpeta raíz): la principal y más completa
   - Versión PHP puro (carpeta "quibdo_seguro html/"): versión alternativa
     con MySQL en lugar de MongoDB

2. La base de datos es MongoDB, NO MySQL. Los modelos usan
   mongodb/laravel-mongodb para las relaciones y queries.

3. El sistema de puntos funciona así:
   - Reportar incidente = +10 puntos
   - Canjear recompensa = -N puntos (según la oferta)

4. Los roles del sistema son: 'normal', 'admin', 'autoridad', 'comercio'
   Cada rol tiene su propio panel y rutas protegidas.

5. El mapa usa Leaflet.js con tiles de OpenStreetMap (gratuito, sin API key).

6. Los códigos QR se generan como strings únicos (no imágenes QR reales aún).
